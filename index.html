Civic Engagement Map

Please contact me to access this private repository.

Final Product

Swarthmore College has an enduring commitment to civic engagement. Civic engagement takes many forms, whether it classes that have a community impact component, grants for student-led social entrepreneurship projects, or politically-engaged research. The diversity and range of these projects is impressive, but it can be difficult to know everything your colleagues are involved in. This was the impetus to create the Civic Engagement Map. The intention was to design a tool that would allow the user to identify colleagues' projects and topic areas on campus and off. The hope was that this will foster connections between actors who identify similarities in their projects, as well as create a centralized database. By fostering connections, we hoped to encourage collaboration and more efficient resource allocation, and methods sharing. The current map prototype allows anyone with a Swarthmore ID to create a profile, log information about their project, and place pins on the map that relate to their project. The map itself is easily filtered by common topics and areas of interest.

I began to be involved in this project in fall, 2019. I worked with Anthony Weed, a programmer in the Swarthmore IT department to continue to develop the main features of the site. My role was mostly front-end development, but supported by a strong grasp on the back-end details. Some of my contributions are as follows:

  • Keyword-search of project cards connected to filtering map pins
  • Enable admin ability to approve projects added by users
  • Highlight regions of interest and high activity on the map
  • Develop user controls : ability to create and delete a project and connect these actions to database
  • Implement user-friendly page-layout to display detials for a single project
    •  My envolvement was not solely technical. I also worked with Nhimesh Ghimire to discuss how broad project goals could be implemented in the design of the application.

    This project allowed me to gain hands-on experience with pgit strofessional tools and to help advance a project that has a palpable impact on the Swarthmore community. If I could do it again, I would have directly consulted students and faculty who intend to use the tool to have a first-hand understanding of their needs and ideas. I have found that it is quite easy to fall into a coding bubble and rely on superiors to interface with the target user community, but I think it would have been valuable to have been in direct communication with peers who plan to use the tool to gain their perspectives not only on broad objectives, but the way these objectives are translated into features.

    Bach Chorale Workshop

    Source Code

    Final Product

    I developed this project as a part of the SPEED program at Swarthmore. Music professor Andrew Hauze had identified that his students studying conducting had a hard time finding adequate tools to be develope their listening skills. For advanced musicians, it is essential to be able to pick out the different melodic and instrumental strains in a piece of music. His students faced a dilemma in developing this skill ; listening to live performances is ideal for hearing different instrumentations, but the experience is not repeatable. Listening to recording is repeatable, but difficult to pick out different strains in the audio. Thus, I worked with him to design an implement a tool that takes a broad database of Bach chorales and allows that user to control the following :
  • Instrumentation
  • Volume
  • Pausing and repeating
  • Visualizing the musical score
  • Entering into a more difficult practice mode that groupd alto and tenor parts in terms of control
    •

    I had the priviledge of writing all of the code for this project, working under the guidance of Ashley Turner and Roberto Vargas. Thus, there were multiple stages in tackling project. Firstly, I had to find or create a database of Bach chorales (which are in the creative commons) and convert the scores into MIDI format if they weren't already. Using the open-source JS library WebAudioFonts, I was able to parse each chorale into Soprano, Alto, Tenor, and Bass parts, and attach controls to each part individually. The second challenge was in procuring the musical scores, matching them to the audio files, and syncing the controls (so that if the audio tempo changed, for instance, then the score highlight should reflect the new tempo). I was able to accomplish this using another open-source library that renders the scores as an svg.

    I had to be flexible with my short-term goals in this project. I had initially thought I would have to implement each aspect from scratch, which poses it's own challenges. After finding appropriate libraries, however, the challenge became to coordinate their functionalities, and to suite them to my specific needs.

    Lastly, after establishing a viable, funcitoning tool, I worked on making it accessible. This entailed making each button, link, and drop-down menu all easily used with a screen-reader, as well as adjusting colors to make sure contrasts are high enough, and making sure that the functionality is clear to the user.
    This project was my first experience taking on the majority of the responsibility, and I learned valuable soft skills such as time management, long and short-term goal setting, and frequent communication with other project participants. I also gained a strong understanding of the website developement through learning-by-doing.
  • HTML, CSS, JS, JQuery
  • Boostrap
  • WebAudioFonts
  • svg score visualizer


    • 1847 African-American Census (Prototype)

    Source Code

    You can view the prototype here. Please contact me to see the final product, as it requires a Swarthmore log-in.

    The special collections visualization project was proposed by Celia Caust-Ellenbogen and Jordan Landes of the Friends Historical Library at Swarthmore College. The library preserved the original manuscript of a census of the 1847 African-American population in Philadelphia, collected by a committee of Quakers. We wanted to revitalize the original online exhibit for the Philadelphia 1847 Census data, in order to present the data in a streamlined, accessible setting for researchers and students. Additionlly, we wanted to add interactivity to the data and present new interpretations of the data through vizualizations. This included a geocoding component to relate the 1847 addresses to contemporary Philadelphia. This would allow for a better understanding of demographic changes over time, as well as envision the history of Philadelphia's black cummunity.
